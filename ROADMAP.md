# Roadmap

## PGM 0.1

PGM 0.1 defines the smallest useful core:

- Core specification
- Reference parser
- Obsidian plugin

The 0.1 goal is interoperability, not feature breadth.

## Future Ideas

The following ideas are intentionally excluded from version 0.1:

- namespaces
- ontology validation
- RDF export
- embedded graph queries
- inference rules

These may be explored after the core syntax and information model have proven stable.

## Non-Goals for 0.1

PGM 0.1 does not define:

- a Markdown replacement
- a graph database
- a query language
- a custom renderer
- a schema language
- an ontology language
- a synchronization protocol

PGM 0.1 only defines how a CommonMark corpus can be interpreted as an openCypher-compatible Property Graph.
